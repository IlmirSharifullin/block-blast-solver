# Block Blast Solver

Инструмент для автоматического решения головоломки Block Blast. Захватывает экран, распознаёт состояние поля и выданные фигуры, находит оптимальное размещение с учётом очистки рядов и столбцов.

## Требования

- Python 3.12+
- [uv](https://docs.astral.sh/uv/) для управления зависимостями
- macOS (захват экрана через `mss` / CoreGraphics)
- Block Blast запущен на экране — через QuickTime Screen Mirroring (iOS) или эмулятор

## Установка

```bash
git clone <repo>
cd block-blast-solver
uv sync
```

## Использование

### Демо-режим (без экрана)

Запуск на встроенном тестовом поле — для проверки работы солвера:

```bash
uv run python main.py --demo
```

### Реальная игра

**Первый запуск — калибровка:**

```bash
uv run python main.py
```

Программа попросит ввести координаты двух областей на экране:
1. Поле 8×8 (верхний левый и нижний правый угол)
2. Панель с тремя фигурами (под полем)

Координаты удобно узнать через **Cmd+Shift+4** на macOS — при выделении области курсор показывает X/Y в пикселях. Настройки сохраняются в `config.json` и при следующих запусках не запрашиваются.

**Повторная калибровка:**

```bash
uv run python main.py --calibrate
```

### Вывод результата

После запуска в терминале печатается пошаговое решение:

```
=== Solution ===
Score: 38

Step 1: place piece 2 at row=8, col=6  (3 cells, clears 2 line(s))
  Rows cleared: [8]
  Cols cleared: [6]
Step 2: place piece 1 at row=5, col=1  (5 cells)
Step 3: place piece 3 at row=3, col=4  (4 cells)
```

Также сохраняются три изображения:

| Файл | Содержимое |
|---|---|
| `debug_grid.png` | Скриншот поля с точками: зелёная = занято, красная = пусто |
| `debug_pieces.png` | Скриншот панели фигур с точками по каждой ячейке |
| `solution.png` | Скриншот поля с цветными оверлеями для каждого хода |

## Как работает солвер

1. **Захват экрана** — `mss` снимает указанные регионы
2. **Парсинг поля** — каждая из 64 ячеек анализируется по насыщенности и яркости в HSV
3. **Парсинг фигур** — в зоне каждой фигуры находятся цветные пиксели, определяется bbox и сетка клеток
4. **Решение** — перебор всех перестановок 3 фигур и всех допустимых позиций с симуляцией очистки рядов
5. **Скоринг**: `+1` за каждую размещённую клетку, `+10` за каждый очищенный ряд или столбец

## Структура проекта

```
block-blast-solver/
├── main.py              # Точка входа (--demo / --calibrate / live)
├── config.json          # Координаты регионов (создаётся при калибровке)
├── src/
│   ├── models.py        # Датаклассы: Grid, Piece, Placement, SolveResult
│   ├── capture.py       # Захват экрана и калибровка
│   ├── detector.py      # OpenCV: парсинг поля и фигур
│   ├── solver.py        # Алгоритм поиска оптимального хода
│   └── visualizer.py    # Вывод в терминал и генерация PNG
```

## Настройка детектора

Если поле или фигуры распознаются некорректно, поправь пороги в [src/detector.py](src/detector.py):

```python
SAT_THRESHOLD = 136    # насыщенность HSV — ниже = больше клеток считается занятыми
BRIGHTNESS_THRESHOLD = 90  # яркость для пустых клеток
```

Ориентируйся на значения из лога — они печатаются для каждой ячейки при парсинге.
